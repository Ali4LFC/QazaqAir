# 📊 Инфраструктура мониторинга QazaqAir

Этот проект представляет собой комплексное, оптимизированное решение для мониторинга всей инфраструктуры QazaqAir на базе Docker. Стек объединяет в себе лучшие инструменты для сбора метрик, визуализации, алертинга и системного мониторинга.

---

## 🏗️ Обзор проекта (Project Overview)

Система мониторинга состоит из следующих ключевых компонентов:
- **Prometheus**: Ядро системы. Собирает и хранит временные ряды метрик.
- **Grafana**: Мощная платформа для визуализации данных с предустановленными дашбордами.
- **Alertmanager**: Обработка алертов и отправка уведомлений (Telegram).
- **Zabbix**: Комплексный мониторинг серверов, сетей и приложений.
- **Nagios**: Классический инструмент для мониторинга состояния сервисов и хостов.
- **Graphite & StatsD**: Сбор и хранение кастомных метрик приложений.
- **Exporters**: Node Exporter (системные метрики) и Blackbox Exporter (доступность эндпоинтов).

> ✨ **Особенности реализации**: Все сервисы оптимизированы (Alpine-образы), лимитированы по ресурсам и очищены от дубликатов конфигураций.

## 🚀 Быстрый запуск (Quick Start)

### 1. Запуск всех сервисов
Перейдите в папку мониторинга и запустите стек:
```bash
cd monitoring
docker-compose up -d
```

### 2. Проверка статуса
Проверьте состояние контейнеров:
```bash
docker-compose ps
```

### 3. Скрипт диагностики
Запустите встроенный скрипт для проверки связей между компонентами:
```bash
chmod +x check_status.sh
./check_status.sh
```

## ⚙️ Руководство по настройке (Configuration Guide)

### 📨 Настройка Telegram (Alertmanager)
Отредактируйте файл `alertmanager/alertmanager.yml`:
1. Вставьте ваш `bot_token`.
2. Вставьте ваш `chat_id`.
3. Примените изменения: `docker-compose restart alertmanager`.

### 🌐 Добавление эндпоинтов (Blackbox Exporter)
Чтобы добавить новый URL для мониторинга доступности, отредактируйте `prometheus/prometheus.yml`:
```yaml
  - job_name: 'blackbox_exporter'
    static_configs:
      - targets:
          - http://google.com
          - http://your-app-service.com  # Добавьте свой адрес здесь
```
После этого выполните: `docker-compose kill -s SIGHUP prometheus` (перезагрузка конфига без остановки).

## 📈 Возможности мониторинга (Monitoring Features)

- **Предустановленные дашборды**: В Grafana автоматически импортирован **Node Exporter Full (ID: 1860)** для детального анализа ресурсов сервера.
- **Базовые алерты**:
  - `InstanceDown`: Срабатывает, если любой из таргетов недоступен более 1 минуты.
  - Уведомления автоматически приходят в Telegram (при условии настройки токена).

## 🤖 Интерактивный Telegram-бот (Interactive Telegram Bot)

В проект включен интерактивный бот, который позволяет не только получать алерты, но и управлять системой.

### 🛠️ Доступные команды
Вы можете управлять ботом с помощью следующих команд:

- `/dashboard` — Генерирует и присылает набор графиков (CPU, RAM, Disk, Network) из Grafana за последние 30 минут.
- `/status` — Краткий отчет о состоянии всех контейнеров (UP/DOWN).
- `/metrics` — Вывод текущих показателей CPU Load и RAM usage из Prometheus.
- `/top_cpu` — ТОП-5 процессов/метрик, потребляющих больше всего ресурсов CPU.
- `/predict_disk` — Интеллектуальный прогноз заполнения диска на основе текущей динамики (на 24 часа).
- `/logs [service_name]` — Последние 20 строк логов конкретного контейнера.
- `/uptime` — Аптайм всей системы и ключевых сервисов.
- `/alerts` — Список текущих активных алертов из Alertmanager.
- `/restart [service_name]` — Перезапуск конкретного контейнера.
- `/render [panel_id]` — Рендеринг конкретной панели из Grafana по её ID.
- `/help` — Показать справку по командам.

### 🔐 Настройка безопасности и визуализации
1.  Отредактируйте файл `telegram_bot.env`.
2.  Добавьте ваш Telegram ID в `ALLOWED_USER_IDS`.
3.  Для работы графиков:
    - Создайте **Service Account** в Grafana (Configuration -> Service Accounts).
    - Выдайте ему права **Viewer**.
    - Сгенерируйте токен и вставьте его в `GRAFANA_API_KEY`.
4.  Перезапустите стек: `docker-compose up -d --build monitor-bot renderer`.

### 📋 Регистрация команд в Telegram
Чтобы команды подсвечивались и были доступны через меню, выполните следующие действия в [@BotFather](https://t.me/BotFather):
1. Отправьте команду `/setcommands`.
2. Выберите вашего бота.
3. Отправьте следующий список команд:
```text
status - Состояние сервисов (UP/DOWN)
metrics - Основные показатели (CPU/RAM)
alerts - Список активных алертов
silence - Временное отключение уведомлений
restart - Перезапуск конкретного сервиса
help - Справка по командам
```

## 🛠️ Обслуживание (Maintenance)

### Просмотр логов
Для просмотра логов конкретного сервиса (например, Zabbix):
```bash
docker-compose logs -f zabbix-server
```

### Обновление конфигурации без простоя
Для Prometheus можно обновить конфиг горячей перезагрузкой:
```bash
docker-compose kill -s SIGHUP prometheus
```

### Ограничение ресурсов
Все лимиты по памяти и ротация логов (max 10MB) настроены в `docker-compose.yml` и вынесены в `.env` для удобства масштабирования.
