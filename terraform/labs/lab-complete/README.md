# Lab Complete: Все 6 лаб в одной инфраструктуре

Эта лабораторная работа объединяет все 6 тем Terraform в единую работающую инфраструктуру.

## Настройки окружения (преднастроено под вашу инфраструктуру)

| Параметр | Значение | Описание |
|----------|----------|----------|
| **Регион AWS** | `eu-central-1` | Frankfurt |
| **AMI** | `ami-0d1ddd83282187d18` | Ubuntu 22.04 LTS |
| **Тип инстанса** | `t3.micro` | Free Tier eligible |
| **VPC CIDR** | `10.0.0.0/16` | Ваша сетевая конфигурация |
| **Public Subnet** | `10.0.0.0/24` | Публичная подсеть |
| **Порты** | 22, 80, 443 | SSH, HTTP, HTTPS |
| **Terraform Provider** | `~> 4.16` | AWS Provider версии 4.16 |

## Как лабы работают вместе

| Lab | Компонент | Роль в инфраструктуре |
|-----|-----------|----------------------|
| **Lab 6** | Data Sources | Получаем AMI Ubuntu и информацию о регионе/AWS аккаунте |
| **Lab 5** | VPC | Создаем изолированную сеть с публичной и приватной подсетями |
| **Lab 5** | Dynamic Blocks | Security Groups с гибкими правилами через for_each |
| **Lab 4** | IAM | Роли и политики для EC2, пользователи для команды |
| **Lab 2** | EC2 | Серверы приложений в приватной сети + Bastion в публичной |
| **Lab 3** | Variables | Окружения (dev/staging/prod) с разными настройками |
| **Lab 3** | Locals | Вычисляемые значения (имена, теги) |
| **Lab 3** | Outputs | Информация о всех ресурсах |
| **Lab 1** | Docker | Jenkins локально для управления деплоем |

## Быстрый старт

### 1. Инициализация

```bash
cd terraform/labs/lab-complete
terraform init
```

### 2. Планирование (dev окружение)

```bash
terraform plan
```

### 3. Применение (dev окружение)

```bash
terraform apply -auto-approve
```

### 4. Production окружение

```bash
terraform apply -var-file=terraform.tfvars.prod -auto-approve
```

### 5. Уничтожение

```bash
terraform destroy -auto-approve
```

## Доступ к ресурсам

После применения Terraform покажет:

- **Jenkins UI**: `http://localhost:8080`
- **Bastion SSH**: `ssh -i dev-private-key.pem ubuntu@<bastion-ip>`
- **Приватные серверы**: Через Bastion:
  ```bash
  ssh -J ubuntu@<bastion-ip> ubuntu@<private-ip>
  ```
- **S3 Bucket**: Артефакты деплоя

## Зависимости ресурсов

```
Data Sources (AMI, AZs)
    ↓
VPC → Subnets → IGW → NAT Gateway
    ↓
Security Groups (Dynamic Blocks)
    ↓
IAM Role + Policy + Instance Profile
    ↓
SSH Key Pair
    ↓
EC2 Instances (в приватной сети)
Bastion (в публичной сети)
    ↓
S3 Bucket
    ↓
Docker Network + Volume
    ↓
Jenkins Container (монтирует SSH ключ)
```

## Проверка работы

1. Откройте Jenkins: `http://localhost:8080`
2. Подключитесь к Bastion: `ssh -i dev-private-key.pem ubuntu@<bastion-ip>`
3. С Bastion подключитесь к приложению:
   ```bash
   curl http://<private-ip>
   # Должно показать: "QazaqAir App - dev - Server X"
   ```
4. Проверьте S3: `aws s3 ls s3://<bucket-name>`

## Очистка

**Важно**: Уничтожьте ресурсы после работы, чтобы избежать расходов!

```bash
terraform destroy -auto-approve
```

Это удалит:
- Все EC2 инстансы
- VPC и все сетевые ресурсы
- IAM роли и пользователей
- S3 бакет
- Docker контейнер и volume
- SSH ключ
